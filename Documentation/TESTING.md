# Testing Guide - My MMO Game\n\n## Overview\n\nThis document provides comprehensive testing guidelines for the MMO project, covering both client-side (Unreal Engine) and server-side (Node.js) testing.\n\n## Quick Start\n\n### Run All Tests\n```bash\ncd Server\nnpm test\n```\n\n### Run Tests in Watch Mode\n```bash\ncd Server\nnpm run test:watch\n```\n\n### Run Tests with Debugging\n```bash\ncd Server\nnpm run test:debug\n```\n\n### Check Code Quality\n```bash\ncd Server\nnpm run lint:check\nnpm run lint  # auto-fix issues\n```\n\n---\n\n## Server-Side Testing (Node.js)\n\n### Test Framework\n- **Framework**: Jest\n- **WebSocket Testing**: socket.io-client\n- **HTTP Testing**: Supertest\n- **Code Coverage**: Istanbul (included with Jest)\n\n### Test Structure\n\n```\nServer/\n├── tests/\n│   ├── setup.js           # Test environment setup\n│   ├── server.test.js     # Socket.IO server tests\n│   ├── health.test.js     # API endpoint tests\n│   └── integration.test.js # Game system integration tests\n├── jest.config.js         # Jest configuration\n└── .eslintrc.json         # ESLint configuration\n```\n\n### Running Specific Tests\n\n**Run only Socket.IO tests:**\n```bash\ncd Server\nnpm test -- server.test.js\n```\n\n**Run only integration tests:**\n```bash\ncd Server\nnpm test -- integration.test.js\n```\n\n**Run with coverage report:**\n```bash\ncd Server\nnpm test -- --coverage\n```\n\n### Test Coverage\n\nCurrent coverage thresholds:\n- **Branches**: 70%\n- **Functions**: 70%\n- **Lines**: 70%\n- **Statements**: 70%\n\nView coverage report:\n```bash\ncd Server\nnpm test\nopen coverage/lcov-report/index.html  # macOS\nstart coverage/lcov-report/index.html # Windows\n```\n\n### Test Categories\n\n#### 1. Socket.IO Server Tests (`server.test.js`)\n\n**Tests included:**\n- ✅ Client connection to server\n- ✅ Player movement broadcasting\n- ✅ Chat message broadcasting\n- ✅ Player disconnection handling\n- ✅ Multiple player tracking\n\n**Run:**\n```bash\nnpm test -- server.test.js\n```\n\n#### 2. API Health Tests (`health.test.js`)\n\n**Tests included:**\n- ✅ Health endpoint returns 200 status\n- ✅ Response contains status message\n- ✅ Response time < 100ms\n\n**Run:**\n```bash\nnpm test -- health.test.js\n```\n\n#### 3. Game System Integration Tests (`integration.test.js`)\n\n**Character System:**\n- ✅ Character initialization with correct stats\n- ✅ Damage calculation\n- ✅ Healing mechanics\n- ✅ Death condition\n\n**Experience & Leveling:**\n- ✅ Experience gain\n- ✅ Level up threshold\n- ✅ Stat bonuses on level up\n\n**Inventory System:**\n- ✅ Empty inventory initialization\n- ✅ Add items to inventory\n- ✅ Calculate total weight\n- ✅ Weight limit enforcement\n- ✅ Remove items\n\n**Equipment System:**\n- ✅ Equip items\n- ✅ Unequip items\n- ✅ Apply equipment stat bonuses\n\n**Run:**\n```bash\nnpm test -- integration.test.js\n```\n\n---\n\n## Client-Side Testing (Unreal Engine)\n\n### In-Editor Testing\n\n#### 1. Character System Tests\n\n**Setup:**\n1. Open project in Unreal Editor\n2. Create a test map or use existing level\n3. Place a CharacterBase actor in the world\n\n**Manual Tests:**\n\n| Test | Steps | Expected |\n|------|-------|----------|\n| **Health** | Play → Check Details Panel | Health = Max Health (100) |\n| **Damage** | Play → Open Console (\\`) → `GetPlayer().TakeDamage 25` | Health = 75 |\n| **Healing** | Play → Console → `GetPlayer().Heal 30` | Health increases |\n| **Death** | Play → Damage until Health = 0 | Character dies, movement disabled |\n| **Level Up** | Play → Console → `GetPlayer().GainExperience 1000` | Level increases |\n\n**Console Commands:**\n```\nstat fps              # Show frame rate\nstat unit             # Frame time breakdown\nstat rendering        # GPU stats\nstat memory           # Memory usage\n```\n\n#### 2. Inventory System Tests\n\n**Blueprint Test Setup:**\n1. Create a Blueprint with InventoryComponent\n2. Add test items via Blueprint\n3. Verify inventory state\n\n**Test Scenarios:**\n\n```\n1. Add Item\n   - Call AddItem() with test item\n   - Verify item appears in inventory array\n\n2. Weight Management\n   - Add multiple items\n   - Monitor GetTotalWeight()\n   - Verify against MaxCarryWeight\n\n3. Equipment\n   - Call EquipItem()\n   - Check EquippedItems map\n   - Verify stat bonuses apply\n```\n\n#### 3. Networking Tests\n\n**Multi-Player Testing:**\n\n1. **Start Server:**\n   ```bash\n   cd Server\n   npm run dev\n   ```\n\n2. **Open Multiple Editors:**\n   - File → New Editor Window (repeat 2-3 times)\n   - Each window simulates a player\n\n3. **Test Scenarios:**\n   - Move Player 1 → Verify Player 2 sees movement\n   - Send chat from Player 1 → All players receive\n   - Disconnect Player 1 → Server shows disconnect event\n\n**Check Server Logs:**\n```\nNew player connected: [socket-id]\nPlayer moved: position data\nPlayer disconnected: [socket-id]\n```\n\n---\n\n## Automated Testing (CI/CD)\n\n### GitHub Actions Pipeline\n\nAutomatically runs on:\n- Push to `main` or `develop` branches\n- Pull requests to `main` or `develop`\n\n**Jobs:**\n1. **Test** - Run Jest tests on Node 16.x, 18.x, 20.x\n2. **Lint** - Check code quality with ESLint\n3. **Build** - Verify project structure and build\n\n**View Results:**\n- Go to repository → Actions tab\n- Click on workflow run\n- View test results and coverage\n\n**Upload Coverage:**\n- Coverage reports automatically uploaded to Codecov\n- View at: `https://codecov.io/gh/callumgram16-byte/my-mmo-game`\n\n---\n\n## Writing New Tests\n\n### Test Structure Template\n\n```javascript\ndescribe('Feature Name', () => {\n  let testData;\n\n  beforeEach(() => {\n    // Setup before each test\n    testData = initializeTestData();\n  });\n\n  afterEach(() => {\n    // Cleanup after each test\n    cleanup();\n  });\n\n  test('should do something specific', () => {\n    // Arrange\n    const input = 'test input';\n\n    // Act\n    const result = functionToTest(input);\n\n    // Assert\n    expect(result).toBe('expected output');\n  });\n});\n```\n\n---\n\n**Last Updated**: 2026-06-25\n**Status**: Active\n
