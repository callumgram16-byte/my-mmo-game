# 🎮 How to Launch Your RPG MMO Game\n\nThis guide will walk you through launching both the server and client for your multiplayer RPG.\n\n---\n\n## 📋 Prerequisites\n\nBefore launching, ensure you have:\n\n### Client Requirements\n- ✅ **Unreal Engine 5.0+** installed ([Download](https://www.epicgames.com/store/en-US/download))\n- ✅ **Visual Studio 2022** with C++ tools\n- ✅ **Git** for version control\n- ✅ Minimum **8GB RAM**, **SSD recommended**\n\n### Server Requirements\n- ✅ **Node.js 16+** installed ([Download](https://nodejs.org/))\n- ✅ **npm** or **yarn** (comes with Node.js)\n- ✅ **Git** for cloning repository\n\n---\n\n## 🚀 Quick Launch (5 Minutes)\n\n### Step 1: Clone the Repository\n```bash\ngit clone https://github.com/callumgram16-byte/my-mmo-game.git\ncd my-mmo-game\n```\n\n### Step 2: Launch the Server\n```bash\ncd Server\nnpm install\nnpm run dev\n```\n\nYou should see:\n```\nMMO Server running on port 3000\nEnvironment: development\n```\n\n### Step 3: Launch the Client\n\n**Option A: In-Editor Play (Fastest)**\n1. Double-click `my-mmo-game.uproject`\n2. Wait for Unreal Editor to open\n3. Click the **Play** button (or press **Alt + P**)\n4. Use **WASD** to move, **Mouse** to look around\n\n**Option B: Packaged Game**\n1. In Unreal Editor: **File → Package Project → Windows (64-bit)**\n2. Choose output folder\n3. Wait for packaging to complete\n4. Run `my-mmo-game.exe` from output folder\n\n---\n\n## 🎯 Step-by-Step Launch Guide\n\n### Phase 1: Initial Setup (One-time)\n\n#### 1.1 Install Dependencies\n\n```bash\n# Clone repository\ngit clone https://github.com/callumgram16-byte/my-mmo-game.git\ncd my-mmo-game\n\n# Install server dependencies\ncd Server\nnpm install\n\n# Go back to project root\ncd ..\n```\n\n#### 1.2 Configure Server (Optional)\n\n```bash\ncd Server\ncp .env.example .env\n\n# Edit .env with your settings (optional for basic testing)\n# Leave defaults for development\n```\n\n#### 1.3 Verify Unreal Engine Setup\n\n1. Open **Epic Games Launcher**\n2. Go to **Library**\n3. Verify **Unreal Engine 5.x** is installed\n4. If not, click **Install**\n\n---\n\n### Phase 2: Launch Server\n\n#### 2.1 Start Development Server\n\n**Windows:**\n```bash\ncd path/to/my-mmo-game/Server\nnpm run dev\n```\n\n**macOS/Linux:**\n```bash\ncd path/to/my-mmo-game/Server\nnpm run dev\n```\n\n#### 2.2 Verify Server is Running\n\nOpen a new terminal:\n```bash\ncurl http://localhost:3000/api/health\n```\n\nExpected response:\n```json\n{\"status\":\"Server is running\"}\n```\n\n✅ **Server is ready!**\n\n---\n\n### Phase 3: Launch Client\n\n#### 3.1 Open Project in Unreal Editor\n\n**Option 1: Direct Open**\n- Navigate to `my-mmo-game` folder\n- Double-click `my-mmo-game.uproject`\n- Unreal Editor will launch automatically\n\n**Option 2: Epic Games Launcher**\n1. Open **Epic Games Launcher**\n2. Go to **Library**\n3. Click on **My Projects**\n4. Find and click **my-mmo-game**\n5. Click **Open**\n\n#### 3.2 First Launch Setup\n\nFirst time opening may take 5-10 minutes:\n- Compiling C++ code\n- Compiling shaders\n- Importing assets\n\n**Wait for:**\n```\nCompiling... [████████████████████] 100%\nShaders compiled successfully\n```\n\n#### 3.3 Play in Editor\n\n1. Once editor is open, click **Play** button (top toolbar)\n2. Or press **Alt + P**\n3. Game window opens\n\n**Controls:**\n- **W** - Move forward\n- **A** - Move left\n- **S** - Move backward\n- **D** - Move right\n- **Space** - Jump\n- **Mouse** - Look around\n- **\\`** (backtick) - Open console\n\n✅ **Client is running!**\n\n---\n\n## 🎮 Testing Multiplayer\n\n### Multi-Player Testing Setup\n\n#### Prerequisites\n- Server running (`npm run dev`)\n- Unreal Editor opened\n\n#### Launch Multiple Clients\n\n**Method 1: Multiple Editor Windows**\n1. Open Unreal Editor\n2. Click **File → New Window → New Editor Window**\n3. Repeat to open 2-3 windows\n4. In each window, click **Play**\n5. You now have multiple players!\n\n**Method 2: Packaged Game Instances**\n1. Package game: **File → Package Project → Windows**\n2. Run `my-mmo-game.exe` multiple times\n3. Each instance is a separate player\n\n#### Test Multiplayer Features\n\n| Feature | How to Test | Expected |\n|---------|------------|----------|\n| **Movement Sync** | Move Player 1 | Player 2 sees movement |\n| **Chat** | Send message | All players receive |\n| **Disconnect** | Close window | Server logs disconnect |\n| **Multi-Player** | Open 3+ instances | All sync together |\n\n---\n\n## 🏃 Running Modes\n\n### Development Mode (Recommended for Testing)\n\n```bash\ncd Server\nnpm run dev\n```\n\n**Features:**\n- Auto-reloads on code changes\n- Verbose logging\n- Slower performance (but easier debugging)\n- Watch mode enabled\n\n### Production Mode\n\n```bash\ncd Server\nnpm start\n```\n\n**Features:**\n- Optimized performance\n- Minimal logging\n- No auto-reload\n- Production-ready\n\n### Debug Mode\n\n```bash\ncd Server\nnpm run test:debug\n```\n\nThen open Chrome and go to `chrome://inspect`\n\n---\n\n## 🧪 Running Tests\n\n### Server Tests\n\n```bash\ncd Server\nnpm test\n```\n\nExpected output:\n```\nTest Suites: 3 passed, 3 total\nTests:       22 passed, 22 total\nTime:        2.456 s\n```\n\n### Test with Coverage\n\n```bash\ncd Server\nnpm test -- --coverage\n```\n\n### Watch Mode (Auto-rerun on changes)\n\n```bash\ncd Server\nnpm run test:watch\n```\n\n---\n\n## 📊 Console Commands (In-Game)\n\nWhile playing, press **\\`** (backtick) to open console:\n\n```\nstat fps              # Show frame rate\nstat unit             # Frame time breakdown\nstat rendering        # GPU stats\nstat memory           # Memory usage\nstat unit memory      # Detailed memory\n```\n\n**Game Commands (if implemented):**\n```\nGainExperience 1000   # Gain 1000 XP\nTakeDamage 25         # Take 25 damage\nHeal 50               # Heal 50 HP\n```\n\n---\n\n## 🐛 Troubleshooting\n\n### Server Won't Start\n\n**Error: \"Port 3000 already in use\"**\n```bash\n# Kill process using port 3000\nlsof -i :3000          # macOS/Linux\nkill -9 <PID>\n\n# Or use different port\nPORT=3001 npm run dev\n```\n\n**Error: \"npm not found\"**\n```bash\n# Install Node.js from https://nodejs.org/\n# Then try again\nnpm run dev\n```\n\n### Unreal Editor Won't Open\n\n**Delete cache folders:**\n```bash\ncd my-mmo-game\nrm -rf Binaries Intermediate Saved  # macOS/Linux\nrmdir /s Binaries Intermediate Saved # Windows\n```\n\n**Regenerate Visual Studio files:**\n1. Right-click `my-mmo-game.uproject`\n2. Select \"Generate Visual Studio project files\"\n3. Double-click `.sln` file to open in Visual Studio\n4. Build → Rebuild Solution\n5. Double-click `.uproject` again\n\n**Verify Unreal Installation:**\n- Open Epic Games Launcher\n- Check if Unreal Engine 5 is installed\n- If not, install it\n\n### Multiplayer Not Syncing\n\n**Verify server is running:**\n```bash\ncurl http://localhost:3000/api/health\n```\n\nIf fails, start server:\n```bash\ncd Server\nnpm run dev\n```\n\n**Check firewall:**\n- Windows: Allow Unreal Editor through firewall\n- macOS: System Preferences → Security & Privacy → Firewall\n- Linux: Open port 3000: `sudo ufw allow 3000`\n\n**Restart editor:**\n1. Close all Unreal Editor windows\n2. Stop server (`Ctrl + C`)\n3. Start server again (`npm run dev`)\n4. Open editor and Play\n\n---\n\n## 📁 File Structure After Launch\n\n```\nmy-mmo-game/\n├── Source/                # C++ source code\n├── Content/              # Game assets (maps, characters, etc.)\n├── Server/\n│   ├── src/\n│   ├── package.json\n│   ├── node_modules/     # Appears after npm install\n│   └── .env              # Configuration (created from .env.example)\n├── Binaries/             # Compiled game (appears after launch)\n├── Intermediate/         # Temporary files (appears during compile)\n├── Saved/                # Game saves (appears after first play)\n└── Documentation/        # Guides and docs\n```\n\n---\n\n## ✅ Checklist Before Playing\n\n- [ ] Node.js installed\n- [ ] Unreal Engine 5 installed\n- [ ] Repository cloned\n- [ ] `npm install` completed in Server folder\n- [ ] Server running without errors\n- [ ] Can connect to `http://localhost:3000/api/health`\n- [ ] Unreal Editor opens without errors\n- [ ] Can press Play and see game window\n- [ ] Can move character with WASD\n- [ ] No critical errors in Output log\n\n---\n\n## 🎯 Next Steps After Launch\n\n1. **Explore the Game**\n   -
